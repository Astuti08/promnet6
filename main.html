Git merupakan salah satu alat (tool) yang digunakan dalam pengembangan perangkat lunak (software). Git adalah Version Control System (VCS) yang digunakan developer untuk mengembangkan software secara bersama-bersama. Fungsi utama dari git yaitu mengatur source code program dengan memberikan tanda baris dan code mana yang ditambah atau diganti. Memudahkan programmer untuk mengetahui perubahan source code daripada harus membuat file baru seperti Program.java, ProgramRevisi.java, ProgramRevisi2.java, ProgramFix.java. Dengan git, kita tidak perlu khawatir code yang dikerjakan akan bentrok dengan cara membuat branch, sehingga mempermudah developer lain untuk mengetahui kendala apa yang dialami developer lain.

Alur dari Git adalah sebagai berikut:

1. mengubah berkas dalam direktori kerja.

2. membawa berkas ke stage.

3. melakukan commit, mengambil berkas yang ada di staging area ke direktori Git

Berikut perintah-perintah dasar git:

  • Git init digunakan untuk membuat repository pada file lokal yang nantinya ada folder.git.
  • Git status digunakan untuk mengetahui status dari repository local.
  • Git add untuk menambahkan file baru pada repository yang dipilih.
  • Git commit digunakan untuk menyimpan perubahan yang dilakukan, tapi tidak ada perubahan pada remote repository.
  • Git push digunakan untuk mengirimkan perubahan file setelah di commit ke remote repository.
  • Git branch untuk melihat seluruh branch yang ada pada repository.
  • Git checkout untuk menukar branch yang aktif dengan branch yang dipilih.
  • Git fetch digunakan hanya akan mengambil perubahan (commit-an) dari remote repository saja tanpa melakukan merge (penggabungan) ke local repository kita.
  • Git pull digunakan untuk mengambil perubahan (commit-an) dari remote repository dan akan langsung digabungkan dengan local repository kita.
  • Git merge digunakan untuk menggabungkan branch yang aktif dan branch yang dipilih.
  • Git clone digunakan untuk membuat salinan repository local.
